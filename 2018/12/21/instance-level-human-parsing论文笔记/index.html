Instance-level Human Parsing 论文笔记

Posted on 2018-12-21

论文: Instance-level Human Parsing via Part Grouping Network
论文地址
代码地址

这是一篇商汤和中山大学 ECCV2018 的一篇论文，我觉得论文主要有两个贡献，首先论文发布了一个新的 crowd instance-level human parsing 的数据集，其次论文提供一个 detecion-free 的 unified 结构 PGN 去解决多人的 human parsing 的问题。

介绍

human parsing 主要做的事情就是识别人的每个部分的语意信息，其是很多任务的基础。目前对于多人的 human parsing 主要是两种做法，一种是 “parsing-by-detection”，也就是先做 detection，再对每一个 detection 的 person 做 human parsing。这样首先需要两个网络通过若干独立目标和阶段来训练，使得训练过程很复杂，同时两个网络的目标函数粒度不同，detection 是粗粒度的 bounding box，而 parsing 是细粒度的 pixel-wise part segmentation，如果用同一个 backbone，会出现一些反常的结果，比如 segmentation 会因为 detection 的原因错误地在 bbox 的外面预测了 part semantic。

在本论文中，作者对这个问题 reformulate，用一个 unified 的网络对两个连续的分割任务进行分组，一个是 part-level pixel-grouping，一个是 instance-level part-grouping。首先 part-level 的分割可以解决人体的各个部分的语义，然后通过 instance-level 的分割再将不同的人分开，这个一致的网络叫做 Part Grouping Network (PGN)。

Part Grouping Network

整体的网络结构如上图所示，使用了 ResNet101 作为 backbone，因为 edge detection 和 semantic segmentation 都是一个 dense recognition 的任务，所以他们可以共享一个相同的 backbone。本文没有使用 multi-scale 的输入，代替的是将中间的 feature map concatenate 在一起，同时中间使用 dilated conv 来保证 feature map 的大小一致。同时本文参考了 PSPNet 的方法，为了融合不同区域的 context 来得到一个全局的 context，使用了 pyramid pooling，经过 (1, 1), (2, 2), (3, 3) 和 (6, 6) 的 avgpooling 得到了 4 个 feature map，然后再通过 conv 和 Upsampling，最后在 concatenate 在一起，如下图所示。

Semantic part segmentation branch

在 semantic 的分支上，直接对 pyramid pooling 之后的结果接一个 1x1 的 conv，然后输出一个 k 个 channels，对应于目标的类别数，包括背景类。

Instance-aware edge detection branch

在 edge detection 这个分支，会对最后三个阶段的 feature map 做一个 atrous spatial pyramid pooling，得到三个输出的 feature map，然后做一个0和1的二分类，边界就是1，非边界就是0。同时也会想上面一个分支一样，将 feature map concatenate 在一起，同时用一个 pyramid pooling 得到不同位置的增强特征，然后再通过一个 1x1 的卷积得到一个 edge 的 feature map，同样做一个二分类。

Refinement branch

除了上面两个分支独立来做，还有另外一个分支联合了 segmentation 和 edge 的结果来做 refinement。用一个 1x1 的 conv 把 segmentation score map 和 edge score map 映射到了高维的 embedding 空间，然后 concat 在一起分别经过两个 pyramid pooling 再次得到 segmentation 和 edge 的 score map，最后的 loss 函数如下

segmentation 用 pixel-wise cross-entropy loss，edge 用 pixel-wise sigmoid binary cross-entropy loss。

Instance partition process

有了 instance-aware edge 和 semantic part segmentation，我们可以把这两个任务组合在一起得到最后 instance-aware human parsing 的结果，这里本文采用的是一种叫 line decoding 的方式。
同时扫描 part segmentation maps 和 edge maps 得到水平和竖直的线。对于水平的线，从左往右，背景的 segmentation 直接被跳过，只有扫描到前景才算，如果击中了边界，这条线就终止，直到又扫描到前景，又开始一条新的线，每条线都有独立的编号，竖直方向和水平方向做法类似。

第二部分我们把竖直的线和水平的线看成一个连通图，同一条直线上的点是连通的，然后我们用 BFS 去遍历整个图，这样就可以得到一个不同的 instance，最后再组合 part segmentation 的结果，就得到了 instance part segmentation 的结果了，非常的直观。

结论

本文一个比较大的创新点是通过 line segmentation 的方式来替代了 detection，从结果来看应该也是可以 work 的，但是不知道这个方法是否真的具有使用价值，因为通过 detection 出来的 person bbox 在做 human parsing 的耗时应该是比在一张大图上去做 segmentation 要快的，同时应该也会更加准确，但是本文并没有对这种类似 mask-RCNN 的方法进行比较。
